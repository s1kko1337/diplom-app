## Общие положения

### Base URL

```
/api
```

### Аутентификация

Все запросы требуют заголовка авторизации:

```
Authorization: Bearer <token>
```

Токен получается через `/api/auth/login`. Срок действия — 8 часов.

### Формат запросов и ответов

- `Content-Type: application/json` для всех запросов с телом
- Все даты — ISO 8601: `YYYY-MM-DD` для дат, `YYYY-MM-DDTHH:mm:ssZ` для datetime
- Ключи JSON — camelCase
- Числа без единиц измерения (единицы хранятся в поле `unit`)

### Пагинация

Коллекционные эндпоинты поддерживают пагинацию:

| Параметр   | Тип    | По умолчанию | Описание                      |
|------------|--------|-------------|-------------------------------|
| `page`     | number | `1`         | Номер страницы (с 1)          |
| `pageSize` | number | `20`        | Записей на странице (макс 100) |

Формат ответа с пагинацией:

```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 47,
    "totalPages": 3
  }
}
```

### Коды HTTP-ответов

| Код | Значение                                          |
|-----|---------------------------------------------------|
| 200 | Успешно                                           |
| 201 | Создано                                           |
| 400 | Некорректный запрос (ошибка валидации)            |
| 401 | Не аутентифицирован                               |
| 403 | Нет прав доступа                                  |
| 404 | Ресурс не найден                                  |
| 409 | Конфликт (недопустимый переход статуса)           |
| 422 | Ошибка бизнес-логики (нельзя выполнить операцию) |
| 500 | Внутренняя ошибка сервера                         |

## Модели данных

### Step (шаг наряда)

```json
{
  "id": "step-1",
  "description": "Проверить состояние тормозных механизмов",
  "requirement": "Максимальная величина отхода колодок 1.7 мм",
  "tools": "Комплект ключей, щуп измерительный",
  "status": "pending",
  "comment": null,
  "startedAt": null,
  "completedAt": null,
  "measurements": [
    {
      "id": "meas-1",
      "description": "Величина отхода тормозных колодок",
      "unit": "мм",
      "norm": "1.7 (не более)",
      "fact": null,
      "passed": null
    }
  ],
  "materials": [
    {
      "id": "mat-1",
      "name": "Масло трансмиссионное",
      "unit": "л",
      "volume": null,
      "brand": null
    }
  ]
}
```

**Допустимые статусы шага:**

| Статус       | Описание                                         |
|--------------|--------------------------------------------------|
| `pending`    | Ожидает выполнения                               |
| `in_progress`| Выполняется (зафиксировано время начала)         |
| `passed`     | Выполнен успешно / соответствует требованиям     |
| `failed`     | Выявлено несоответствие требованиям              |
| `skipped`    | Пропущен (с обоснованием в поле `comment`)       |

**Допустимые переходы статусов шага:**

```
pending → in_progress  (через POST .../start)
pending → skipped
in_progress → passed
in_progress → failed
in_progress → skipped
```

### Order (наряд на ТО)

```json
{
  "id": "ТО-004",
  "equipmentId": "БУР-12",
  "type": "TO-1",
  "status": "in_progress",
  "assignedTo": "Петров С.В.",
  "scheduledDate": "2026-03-22",
  "startedAt": "2026-03-22T07:00:00Z",
  "completedAt": null,
  "operatingHoursAtStart": 4850,
  "steps": [...],
  "executors": [
    { "name": "Петров С.В.", "position": "Механик" }
  ],
  "acceptedBy": {
    "name": "Козлов Д.А.",
    "position": "Мастер"
  },
  "remarks": null,
  "createdAt": "2026-03-20T10:00:00Z",
  "updatedAt": "2026-03-22T09:30:00Z"
}
```

**Допустимые статусы наряда:**

| Статус        | Описание                                              |
|---------------|-------------------------------------------------------|
| `draft`       | Черновик — наряд создан, не начат                     |
| `in_progress` | Выполняется                                           |
| `completed`   | Завершён — ожидает приёмки мастером                   |
| `accepted`    | Принят мастером, допущен к эксплуатации               |
| `rejected`    | Возвращён на доработку                                |
| `cancelled`   | Отменён                                               |

**Допустимые типы ТО:**

| Тип    | Описание                         |
|--------|----------------------------------|
| `TO-1` | Техническое обслуживание № 1     |
| `TO-2` | Техническое обслуживание № 2     |
| `TO-3` | Техническое обслуживание № 3     |
| `EO`   | Ежесменное обслуживание          |
| `TR`   | Текущий ремонт                   |

**Допустимые переходы статусов наряда:**

```
draft → in_progress   (начало выполнения)
draft → cancelled
in_progress → completed  (исполнитель завершил)
in_progress → cancelled
completed → accepted     (мастер принял)
completed → rejected     (мастер вернул на доработку)
rejected → in_progress   (возобновлена работа)
```

## Наряды (Orders)

### GET /api/maintenance/orders

Получить список нарядов с фильтрацией.

**Query-параметры:**

| Параметр      | Тип    | Обязательный | Описание                                    |
|---------------|--------|-------------|---------------------------------------------|
| `equipmentId` | string | нет         | Фильтр по идентификатору оборудования       |
| `type`        | string | нет         | Фильтр по типу ТО (`TO-1`, `TO-2`, и т.д.) |
| `status`      | string | нет         | Фильтр по статусу наряда                    |
| `assignedTo`  | string | нет         | Фильтр по имени исполнителя                 |
| `dateFrom`    | string | нет         | Дата начала периода (`YYYY-MM-DD`)          |
| `dateTo`      | string | нет         | Дата окончания периода (`YYYY-MM-DD`)       |
| `page`        | number | нет         | Страница (по умолчанию `1`)                 |
| `pageSize`    | number | нет         | Размер страницы (по умолчанию `20`)         |

**Ответ 200:**

```json
{
  "data": [
    {
      "id": "ТО-004",
      "equipmentId": "БУР-12",
      "type": "TO-1",
      "status": "in_progress",
      "assignedTo": "Петров С.В.",
      "scheduledDate": "2026-03-22",
      "startedAt": "2026-03-22T07:00:00Z",
      "completedAt": null,
      "operatingHoursAtStart": 4850,
      "executors": [
        { "name": "Петров С.В.", "position": "Механик" }
      ],
      "acceptedBy": null,
      "remarks": null,
      "createdAt": "2026-03-20T10:00:00Z",
      "updatedAt": "2026-03-22T09:30:00Z"
```

### PATCH /api/maintenance/orders/:id/status

Изменить статус наряда. Используется для перевода наряда по жизненному циклу (начало, завершение, приёмка, отклонение).

**Path-параметры:**

| Параметр | Тип    | Описание             |
|----------|--------|----------------------|
| `id`     | string | Идентификатор наряда |

**Тело запроса:**

```json
{
  "status": "accepted",
  "operatingHours": 4855,
  "reviewedBy": "Козлов Д.А.",
  "returnReason": null
}
```

**Схема тела запроса:**

| Поле             | Тип    | Обязательный | Описание                                                                                   |
|------------------|--------|-------------|--------------------------------------------------------------------------------------------|
| `status`         | string | да          | Новый статус наряда                                                                        |
| `operatingHours` | number | условно     | Моточасы на момент завершения. Обязателен при переходе `in_progress → completed`          |
| `reviewedBy`     | string | условно     | ФИО мастера, принявшего/отклонившего наряд. Обязателен при переходе `completed → accepted/rejected` |
| `returnReason`   | string | условно     | Причина возврата на доработку. Обязателен при переходе `completed → rejected`             |

**Ответ 200:**

```json
{
  "id": "ТО-004",
  "status": "accepted",
  "acceptedBy": {
    "name": "Козлов Д.А.",
    "position": "Мастер"
  },
  "completedAt": "2026-03-22T14:00:00Z",
  "operatingHoursAtStart": 4850,
  "updatedAt": "2026-03-22T14:05:00Z"
}
```

**Ошибки:**

| Код | Причина                                                        |
|-----|----------------------------------------------------------------|
| 400 | Отсутствуют обязательные для данного перехода поля            |
| 401 | Не аутентифицирован                                            |
| 403 | Нет прав на данное действие (например, приёмку может делать только мастер) |
| 404 | Наряд не найден                                                |
| 409 | Переход статуса недопустим из текущего состояния              |
| 422 | Не все шаги наряда завершены — нельзя перевести в `completed`  |

### POST /api/maintenance/orders/:id/steps/:stepId/start

Зафиксировать начало выполнения шага. Устанавливает `startedAt` и переводит шаг в статус `in_progress`.

**Path-параметры:**

| Параметр | Тип    | Описание             |
|----------|--------|----------------------|

| `stepId` | string | Идентификатор шага   |

**Тело запроса:** пустой объект `{}`

**Ответ 200:**


## Журнал технического состояния (Journal)

Журнал соответствует «Приложению Б» — Журналу технического состояния оборудования.

### GET /api/journal

Получить записи журнала с фильтрацией.

**Query-параметры:**

| Параметр      | Тип    | Обязательный | Описание                                     |
|---------------|--------|-------------|----------------------------------------------|
| `equipmentId` | string | нет         | Фильтр по идентификатору оборудования        |
| `dateFrom`    | string | нет         | Начало периода (`YYYY-MM-DD`)                |
| `dateTo`      | string | нет         | Конец периода (`YYYY-MM-DD`)                 |
| `orderId`     | string | нет         | Фильтр по связанному наряду                  |
| `page`        | number | нет         | Страница (по умолчанию `1`)                  |
| `pageSize`    | number | нет         | Размер страницы (по умолчанию `20`)          |

```json
    "page": 1,
    "pageSize": 20,
    "total": 15,
    "totalPages": 1
  }
}
```

### POST /api/journal

Создать новую запись в журнале технического состояния.

**Тело запроса:**

```json
{
  "equipmentId": "БУР-12",
  "date": "2026-03-22",
  "time": "07:00",
  "description": "Проведено ТО-1, наряд ТО-004",
  "clearance": "Допущен к эксплуатации",
  "orderId": "ТО-004"
}
```

**Схема тела запроса:**

| Поле          | Тип    | Обязательный | Ограничения                                           |
|---------------|--------|-------------|-------------------------------------------------------|
| `equipmentId` | string | да          | Существующий идентификатор оборудования               |
| `date`        | string | да          | `YYYY-MM-DD`                                          |
| `time`        | string | нет         | `HH:mm` (UTC). Если не передано — текущее время       |
| `description` | string | да          | Непустая строка, описание записи                      |
| `clearance`   | string | да          | Допуск к эксплуатации: `Допущен к эксплуатации` / `Не допущен к эксплуатации` / `Допущен с ограничениями` |
| `orderId`     | string | нет         | Идентификатор связанного наряда                       |

**Ответ 201:**

```json
{
  "id": "journal-16",
  "equipmentId": "БУР-12",
  "date": "2026-03-22",
  "time": "07:00",
  "description": "Проведено ТО-1, наряд ТО-004",
  "clearance": "Допущен к эксплуатации",
  "authorName": "Козлов Д.А.",
  "orderId": "ТО-004",
  "createdAt": "2026-03-22T07:05:00Z"
}
```

> Поле `authorName` заполняется автоматически из данных аутентифицированного пользователя.

**Ошибки:**

| Код | Причина                                                     |
|-----|-------------------------------------------------------------|
| 400 | Некорректное тело запроса или отсутствуют обязательные поля |
| 401 | Не аутентифицирован                                         |
| 403 | Нет прав на запись в журнал                                 |
| 404 | Оборудование или указанный `orderId` не найдены             |
| 422 | Недопустимое значение поля `clearance`                      |

## Коды ошибок

### Формат ответа при ошибке

```json
{
  "error": {
    "code": "INVALID_STATUS_TRANSITION",
    "message": "Переход из статуса 'accepted' в статус 'in_progress' недопустим",
    "details": {
      "currentStatus": "accepted",
      "requestedStatus": "in_progress",
      "allowedTransitions": []
    }
  }
}
```

### Справочник кодов ошибок

| Код                         | HTTP | Описание                                                     |
|-----------------------------|------|--------------------------------------------------------------|
| `VALIDATION_ERROR`          | 400  | Ошибка валидации тела запроса или параметров                 |
| `UNAUTHORIZED`              | 401  | Отсутствует или недействителен токен авторизации             |
| `FORBIDDEN`                 | 403  | Недостаточно прав для выполнения операции                    |
| `NOT_FOUND`                 | 404  | Запрошенный ресурс не найден                                 |
| `INVALID_STATUS_TRANSITION` | 409  | Запрошенный переход статуса недопустим                       |
| `ACTIVE_ORDER_EXISTS`       | 422  | У оборудования уже есть активный наряд данного типа          |
| `STEPS_NOT_COMPLETED`       | 422  | Не все шаги наряда завершены — нельзя перевести в `completed`|
| `ORDER_NOT_FINALIZED`       | 422  | Наряд не завершён — документ сформировать нельзя             |
| `STEP_NOT_STARTED`          | 409  | Шаг ещё не начат — сначала вызовите `.../start`             |
| `PDF_NOT_IMPLEMENTED`       | 501  | Генерация PDF не реализована                                 |
| `INTERNAL_ERROR`            | 500  | Внутренняя ошибка сервера                                    |
